Алгоритм для поиска

Есть объекты:

var head = {
  glasses: 1
};

var table = {
  pen: 3
};

var bed = {
  sheet: 1,
  pillow: 2
};

var pockets = {
  money: 2000
};

Задание состоит из двух частей:
Присвойте объектам ссылки __proto__ так, чтобы любой поиск чего-либо шёл по алгоритму pockets -> bed -> table -> head.

То есть pockets.pen == 3, bed.glasses == 1, но table.money == undefined.

После этого ответьте на вопрос, как быстрее искать glasses: обращением к pockets.glasses или head.glasses? Попробуйте протестировать.

Ответ: 1. Ответ по цепочке 'pockets -> bed -> table -> head' во внутреннем скрипте
2. Логично что head.glasses найдет быстрее и с 1-го раза. Чего не скажешь о получении через pockets, когда прийдеться искать в 4 итерации. Но, литература пишет, что современные браузеры запоминают откуда берется свойство, и при следуюших обращениях обращаются сразу к корректному прототипу. Поэтому с точки зрения пользователя, разницу в быстродействии он врядли заметит.
